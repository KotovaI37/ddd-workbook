**I. Тестовая стратегия для каждого контекста.**

**1\. Паттерн бизнес-логики → Архитектура → Стратегия тестирования.**

_Биллинг, Читалка - Модель предметной области (Domain Model):_ домен содержит сложные инварианты, которые легко тестировать изолированно, а адаптеры и инфраструктура тестируются интеграционно. Будут использоваться шестиугольная(Hexagonal) архитектура (для портов и адаптеров) и Пирамида тестирования (много unit).

_DRM -Event Sourced Domain Model_: события и агрегаты идеальны для unit-тестов. Проекции тестируются интеграционно. Рекомендуется использование шестиугольной архитектуры и CQRS. В качестве стратегии тестирования будет использоваться Пирамида тестирования.

_Персонализация, Уведомления, Модерация - Транзакционный сценарий (Transaction Script):_ логика простая, основные риски — в интеграции с БД и внешними системами. Используется слоистая (Layered) архитектура. Стратегия тестирования - Перевернутая пирамида (больше интеграционных тестов).

_Каталог, Пользователи - Активная запись (Active Record):_ основная логика — в БД (валидации, constraints), тестируем через интеграционные тесты. Архитектура – Layered. Стратегия тестирования – Перевернутая пирамида.

**2\. Перечень тест-кейсов (без кода).**

1) _DRM (пирамида):_

**Уровень Unit** – проверяется агрегат License

Тест-кейсы:

\- Выдача лицензии создает событие LicenseIssued

\- Регистрация устройства в пределах лимита успешна

\- Регистрация 6-го устройства вызывает DeviceLimitReached

\- Отзыв лицензии меняет статус

\- Истекшая лицензия не дает доступ

**Уровень Integration -** Проверяется репозиторий Event Store

Тест-кейсы:

\- Сохранение событий в БД

\- Восстановление агрегата из событий

\- Загрузка снэпшота (снимок состояния данных)

Проверка проекции Read Model.

Тест-кейсы:

\- Построение Read Model из событий

\- Обновление при новых событиях

**Уровень E2E** – проверка сквозного сценария – Тест-кейс: Покупка книги → выдача лицензии → чтение на 3 устройствах.

2) _Биллинг (пирамида):_

**Уровень Unit** – проверка агрегата Subscription

Тест-кейсы:

\- Создание подписки с пробным периодом

\- Активация после первого платежа

\- Списание средств за новый период

\- Отмена подписки (доступ до конца периода)

\- Попытка списания дважды за период → ошибка

Проверка Value Object Money

Тест-кейсы:

\- Сложение денежных сумм

\- Сравнение

\- Валидация (неотрицательность)

**Уровень Integration** – проверка репозитория Subscription

Тест-кейсы:

\- Сохранение и загрузка агрегата

\- Поиск по userId

Проверка адаптер PaymentGateway

Тест-кейсы:

\- Вызов внешнего API

\- Обработка ответов (success/failure)

\- Таймауты и retry

**Уровень E2E** – проверка сквозного сценария - Тест-кейс: Пользователь покупает книгу → платеж успешен → лицензия выдана

3) _Персонализация (Перевернутая пирамида):_

**Уровень Integration** – проверяется обработка событий

Тест-кейсы:

\- Получение BookPublished → обновление индекса

\- Получение ProgressUpdated → обновление профиля

Проверка генерации рекомендаций

Тест-кейсы:

\- Запрос рекомендаций для пользователя с историей

\- Пустые рекомендации для нового пользователя

Тестирование хранилища

Тест-кейсы:

\- Сохранение событий в БД

\- Кэширование в Redis

**Уровень Unit** – проверка алгоритмов

Тест-кейсы:

\- Расчет похожести книг

\- Ранжирование результатов

**Уровень E2E** – проверка сквозного сценария – Тест-кейс: Пользователь дочитал книгу → рекомендации обновились → отобразились в UI.
