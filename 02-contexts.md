_Контекст 1: Управление каталогом (Catalog Management)_

|     |     |
| --- | --- |
| Поле | Значение |
| Название контекста | Управление каталогом |
| Назначение | Хранение и предоставление структурированной информации о книгах (метаданных), управление библиотекой произведений. |
| Основные термины<br><br>единого языка (UL) | Книга, Автор, Жанр, Серия, Тег, ISBN, Аннотация, Обложка, Год издания, Издательство, Правообладатель, Язык, Формат (текст, аудио), Возрастное ограничение. |
| Ключевые сущности /<br><br>агрегаты | Агрегат «Книга», сущность «Автор», сущность «Жанр», Value Object «ISBN», Value Object «Метаданные» |
| Входящие данные /<br><br>запросы | Запрос информации о книге (по ID, по ISBN), поиск/фильтрация книг (по жанру, автору), команда на добавление/обновление книги (от администратора/импорта от издательств) |
| Исходящие данные / события | КнигаДобавлена, КнигаОбновлена, МетаданныеИзменены (публикуются для обновления поисковых индексов и уведомления других контекстов) |
| Политика согласованности | Строгая согласованность внутри контекста (книга или есть, или её нет). Кэширование данных для чтения приветствуется |
| Поддомен(ы) | Каталог и метаданные (Generic / Core) |

_Контекст 2: Персонализация и рекомендации (Personalization)_

| Основные термины<br><br>единого языка (UL) | Пользователь, Событие (Просмотр, Чтение, Покупка, Оценка), Вес, Рекомендация, Персональная подборка, Похожие книги, Вам может понравиться, Коллаборативная фильтрация. |

| Поддомен(ы) | Рекомендательная система (Core) |

_Контекст 3: Читалка (Reader)_

|     |     |
| --- | --- |
| Поле | Значение |
| Название контекста | Читалка |
| Назначение | Предоставление интерфейса для чтения книг, управления библиотекой пользователя (избранное), ведения заметок и сохранения прогресса. |
| Основные термины<br><br>единого языка (UL) | Полка (виртуальная), Избранное, Закладка, Прогресс чтения, Позиция чтения, Заметка, Выделение, Цитата, Текст книги, Формат (FB2, EPUB), Ридер, Тема оформления, Шрифт. |
| Ключевые сущности /<br><br>агрегаты | Агрегат «КнигаПользователя» (связка пользователя и книги с прогрессом), сущность «Закладка», сущность «Заметка», Value Object «Позиция» |
| Входящие данные /<br><br>запросы | Запрос на открытие книги (файл/поток), сохранение прогресса, создание закладки/заметки, запрос содержимого полок пользователя |
| Исходящие данные / события | ПрогрессОбновлён, ЗакладкаДобавлена, ЗаметкаДобавлена (отправляются в контекст Персонализации и Аналитики) |
| Политика согласованности | Строгая для операций с заметками и прогрессом (важно не потерять). Конечная — для обновления списков книг. |
| Поддомен(ы) | Читалка (ридер) (Core) |

_Контекст 4: Управление доступом (Access & DRM)_

|     |     |
| --- | --- |
| Поле | Значение |
| Название контекста | Управление доступом |
| Назначение | Проверка прав пользователя на книгу, управление лицензиями и ключами DRM, выдача токенов для чтения. |
| Основные термины<br><br>единого языка (UL) | Право доступа, Лицензия, Ключ DRM, Токен скачивания, Владелец, Срок действия, Пробный период, Ограничение (количество устройств). |
| Ключевые сущности /<br><br>агрегаты | Агрегат «Лицензия» (связывает пользователя и книгу), Value Object «DRM-ключ» |
| Входящие данные /<br><br>запросы | Запрос на проверку доступа (из Читалки), событие КнигаКуплена (из контекста Биллинга), запрос на генерацию токена для скачивания |
| Исходящие данные / события | ДоступПредоставлен / ДоступЗапрещён (ответ на запрос), ЛицензияАктивирована, ДостигнутЛимитУстройств |
| Политика согласованности | Строгая согласованность при проверке прав. Ошибка здесь приведет либо к пиратству, либо к блокировке легального пользователя |
| Поддомен(ы) | Управление цифровыми правами (DRM) (Core) |
